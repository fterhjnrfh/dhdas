# GPU 加速验证指南

本文档将指导您如何验证 SkiaSharp GPU 加速是否在您的项目中正常工作。

## 1. 验证方法概述

有多种方法可以验证 GPU 加速是否启用：

### 1.1 监控工具法
- 使用 Windows 任务管理器监控 GPU 使用率
- 使用第三方工具如 MSI Afterburner 监控 GPU 性能

### 1.2 代码调试法
- 在代码中添加调试输出来确认 GPU 上下文创建
- 比较 GPU 和 CPU 渲染的性能差异

### 1.3 视觉效果法
- 观察波形渲染的流畅度
- 比较大量波形同时渲染时的表现

## 2. 具体验证步骤

### 2.1 使用任务管理器监控 GPU 使用率

1. 打开 Windows 任务管理器 (Ctrl+Shift+Esc)
2. 切换到"性能"选项卡
3. 点击"GPU"查看 GPU 使用情况
4. 运行您的应用程序
5. 观察 GPU 使用率是否随着波形渲染而上升

### 2.2 使用代码调试输出验证

在 `GPUAcceleratedWaveformCanvas.cs` 的 `OnOpenGlInit` 方法中，您会看到如下调试输出：

```csharp
System.Diagnostics.Debug.WriteLine($"GPU 上下文已成功创建 - 最大纹理尺寸: {maxTextureSize}");
System.Diagnostics.Debug.WriteLine("GPU 加速渲染已启用");
```

如果看到这些输出，说明 GPU 加速已经启用。

### 2.3 性能测试验证

使用我们提供的测试控件：

1. 点击"添加50个波形"按钮添加大量波形
2. 点击"测试性能"按钮进行性能测试
3. 观察性能结果显示的平均渲染时间
   - 如果平均时间 < 10ms，很可能使用了 GPU 加速
   - 如果平均时间在 10-50ms 之间，可能使用 GPU 加速或 CPU 渲染
   - 如果平均时间 > 50ms，很可能是 CPU 渲染

## 3. 预期效果

### 3.1 启用 GPU 加速后的效果
- GPU 使用率明显上升（在任务管理器中观察）
- 大量波形渲染时依然保持流畅
- 平均渲染时间显著降低
- CPU 使用率相对较低

### 3.2 未启用 GPU 加速的效果
- GPU 使用率几乎不变或轻微上升
- 大量波形渲染时出现明显卡顿
- 平均渲染时间较长
- CPU 使用率较高

## 4. 故障排除

### 4.1 如果 GPU 加速未启用

1. 检查系统是否支持 OpenGL 3.2+
2. 确认显卡驱动已正确安装并更新到最新版本
3. 检查项目是否正确引用了 SkiaSharp 相关包
4. 确认代码中是否有错误阻止 GPU 上下文创建

### 4.2 如果性能没有明显改善

1. 检查是否真的启用了 GPU 加速（通过调试输出）
2. 确认渲染的数据量足够大以体现 GPU 加速的优势
3. 检查是否有其他瓶颈（如数据生成、内存分配等）

## 5. 进一步优化建议

1. **批处理渲染**：尽可能将多个波形合并到一次渲染调用中
2. **纹理缓存**：对于静态或变化较少的元素，使用纹理缓存
3. **减少状态切换**：尽量减少 GPU 状态切换次数
4. **使用实例化渲染**：对于相同或相似的波形，使用实例化渲染技术

通过以上方法，您应该能够清楚地验证 GPU 加速是否在您的项目中正常工作。
